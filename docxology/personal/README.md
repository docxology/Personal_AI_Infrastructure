# Personal Content

Your personal content folder - plans, notes, customizations, and user-specific materials.

---

## Purpose

The `personal/` folder is for content that is:

- **Personal to you** - Plans, goals, notes
- **User-specific** - Custom workflows, preferences
- **Private** - Not intended for public sharing
- **Evolving** - Grows with your needs

---

## What Belongs Here

### Plans & Goals

- Life plans (like `LIFE_PLAN_Q1_2026.md` in root for quick access)
- Project plans
- Goal tracking
- Personal roadmaps

### Custom Workflows

- Personal automation scripts
- Custom skill definitions
- User-specific configurations
- Workflow documentation

### Notes & Documentation

- Personal notes
- Meeting summaries
- Research notes
- Learning logs

### Customizations

- Personal PAI extensions
- Custom templates
- User-specific tools
- Private configurations

---

## Organization Suggestions

Organize by topic or time:

```
personal/
├── plans/
│   ├── 2026/
│   └── 2027/
├── workflows/
│   ├── research.md
│   └── writing.md
├── notes/
│   └── meetings/
└── custom/
    └── skills/
```

Or keep it flat if you prefer:

```
personal/
├── research-workflow.md
├── writing-process.md
└── project-ideas.md
```

---

## Privacy Considerations

**Important:** This folder may contain personal information.

- Review before committing to public repositories
- Consider adding to `.gitignore` if needed
- Use git filters for sensitive content
- Keep backups of important personal content

---

## Integration with PAI

Your personal content can integrate with PAI:

- **Custom Skills** - Reference personal workflows
- **CORE Skill** - Link to personal plans
- **History System** - Personal notes captured automatically
- **Scripts** - Personal automation tools

---

## Quick Access

For frequently accessed personal content, consider:

- Keeping in root (like `LIFE_PLAN_Q1_2026.md`)
- Creating symlinks
- Adding to CORE skill context
- Referencing in workflows

---

## Evolution

This folder grows with your needs:

- Start simple (flat structure)
- Organize as content grows
- Create subfolders when needed
- Archive old content periodically
